CACHE ARCHITECTURE
┌────────────────────────────────────────────────────────────────────────┐
│                                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                      GitHub Actions Cache                       │  │
│  │                                                                 │  │
│  │  📦 npm dependencies       (~100-200 MB)  [by setup-node]      │  │
│  │  🌐 Playwright browsers    (~200-300 MB)  [NEW]                │  │
│  │  🏗️  SvelteKit build        (~50-150 MB)   [NEW]                │  │
│  │  🦀 Rust/Cargo deps        (~500-1000 MB) [existing]           │  │
│  │                                                                 │  │
│  │  Cache Keys:                                                    │  │
│  │  • Based on file hashes (package-lock.json, Cargo.lock, src)   │  │
│  │  • Smart invalidation when dependencies or source changes       │  │
│  │  • Restore keys for partial cache hits                          │  │
│  └─────────────────────────────────────────────────────────────────┘  │
│                                                                        │
│  Cache Flow:                                                           │
│  1. Check for exact cache key match                                   │
│  2. If match: Restore cache (seconds)                                 │
│  3. If no match: Try restore key patterns (partial restore)           │
│  4. If no restore: Full download/build (populate cache)               │
│  5. After workflow: Save cache for future runs                        │
└────────────────────────────────────────────────────────────────────────┘
